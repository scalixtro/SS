Al tener un mayor número de entradas para un sistema difuso, el número de reglas crece de forma exponencial. Esto reduce la eficiencia computacional, además de que hace menos entendible nuestro sistema. Otra consecuencia de tener un número muy grande de reglas es que se pierde generalización y nuestro sistema de inferencia sólo funcionará correctamente con los datos de ejemplo con el que lo probamos.
Para evitar algunos de estos problemas se puede implementar un sistema de inferencia difuso como un árbol de otros sistemas de inferencia interconectados. En este tipo de estructura tenemos sistemas de inferencia de bajo nivel cuyas salidas sirven de entradas para otros sistemas de inferencia. Tipos de estructuras

Por otro lado, si usáramos un FIS monolítico para las cuatro entradas, tendríamos reglas, con lo que podemos concluir que los árboles difusos disminuyen la cantidad total de reglas que debemos utilizar.
Las entradas para cada nivel se determinan de acuerdo con la importancia que tienen en la salida. Aquellas entradas que consideremos más importantes se uusan en los niveles más bajos, y mientras subimos en el árbol la importancia de las entradas va disminuyendo.

En esta estructura las entradas se agrupan para ciertos resultados específicos. Por ejemplo, tenemos un robot autónomo que combina tareas de evitar obstáculos y alcanzar objetivos para navegar sin colisionar. Para ello, podemos utilizar cuatro entradas:
  • distancia al obstáculo más cercano.
  • ángulo del obstáculo más cercano.
  • distancia al objetivo.
  • ángulo del objetivo.

Una variación en la estructura

Se puede variar la estructura de nuestro árbol agregado utilizando como nodo final una suma entre las salidas de nuestros sistemas de inferencia. Esta estructura es conocida como estructura paralela.

Estructura en cascada o combinada

En una estructura en cascada o combinada se utilizan las estructuras incremental y agregada en un árbol. Esta funciona para sistemas que tienen entradas tanto correlacionadas (como el caso del ángulo y distancia para diferentes variables del robot) como no correlacionadas.
